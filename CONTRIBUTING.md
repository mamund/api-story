# Contributing

Thanks for helping improve this API Stories collection.

## What to contribute

- Clarifications to the specification or guide
- Additional examples
- Improvements to templates
- Corrections to typos, formatting, and consistency
- Additional anti-patterns with concrete symptoms and corrections

## How to contribute

1. Fork the repository.
2. Create a branch with a descriptive name.
3. Make focused changes that are easy to review.
4. Open a pull request.

## Writing guidelines

- Use sentence-case headings.
- Avoid transport details unless the document explicitly calls for them.
- Prefer clear, declarative statements.
- Keep lists and definitions consistent across docs.

## Review expectations

Pull requests should:

- Keep the reading order coherent
- Avoid duplicating content across multiple documents without a reason
- Include updates to examples or templates when the spec changes
